# Настройка профилей с запрещающим все правилом Файрвола

## INPUT

### Разрешить прямые подключения к прокси

Выполните действия:

1\. В разделе **Профили безопасности -> Контроль приложений** создайте профиль, который разрешает только HTTP и TLS, а остальное запрещает:

![](/.gitbook/assets/application-control24.png)

2\. В разделе **Сервисы -> Прокси** включите опцию **Разрешить прямые подключения к прокси** и выберите порт.

3\. Если Ideco NGFW не является шлюзом по умолчанию для клиентов сети, настройте прямой прокси на сетевых устройствах локальной сети, веб-трафик которых нужно пропускать через прокси. Для этого укажите порт, выбранный на стороне NGFW, в браузере на устройствах пользователей:

![](/.gitbook/assets/proxy9.png)

Если Ideco NGFW указан в настройках сетевых подключений как шлюз, этот шаг можно пропустить.

4\. Создайте разрешающее правило INPUT, заполнив поля:

![](/.gitbook/assets/firewall3.png)

* **Протокол** - протокол транспортного уровня (TCP, UDP). В правиле можно выбрать только один протокол, при необходимости создайте два правила;
* **Порт** - порт, выбранный в настройках **Прокси**;
* **Профили безопасности** - профиль, созданный в шаге 1.

Включите правило и нажмите **Добавить**.

5\. Создайте запрещающее весь трафик правило Файрвола. В таблице INPUT поместите созданное разрешающее правило выше запрещающего. До того, как включить запрещающее правило, убедитесь, что включено разрешающее правило.

При потере доступа к личному кабинету администратора Ideco NGFW воспользуйтесь опцией **Включить режим 'Разрешить интернет всем'** в локальном меню и добавьте в созданное правило порт 8443.

### Разрешить подключение к личному кабинету администратора Ideco NGFW из локальной сети

Выполните действия:

1\. В разделе **Профили безопасности -> Контроль приложений** создайте профиль, который разрешает только HTTP и TLS, а остальное запрещает:

![](/.gitbook/assets/application-control24.png)

2\. Создайте разрешающее правило INPUT, заполнив поля:

![](/.gitbook/assets/firewall4.png)

* **Протокол** - протокол транспортного уровня (TCP, UDP). В правиле можно выбрать только один протокол, при необходимости создайте два правила;
* **Порт** - 8443;
* **Зона источника** - можно выбрать только один источник, для каждого источника создается отдельное правило. Выберите значение **Локальный интерфейс**, чтобы разрешить доступ из локальной сети.

Включите правило и нажмите **Добавить**.

3\. Если веб-интерфейс Ideco NGFW доступен не по IP-адресу, а по домену, необходимо в таблице INPUT создать правила для протоколов TCP и UDP, разрешающие подключение из локальной сети на порт 53. Это позволит Ideco NGFW резолвить DNS-запросы при наличии запрещающего правила в INPUT:

![](/.gitbook/assets/firewall9.png)

4\. Создайте запрещающее весь трафик правило Файрвола. В таблице INPUT поместите созданное разрешающее правило выше запрещающего. До того, как включить запрещающее правило, убедитесь, что включено разрешающее правило.

## FORWARD

### Разрешить доступ к почтовому клиенту Thunderbird

Выполните действия:

1\. В настройках сети на устройстве пользователя укажите IP-адрес локального интерфейса Ideco NGFW в качестве шлюза.

2\. В разделе **Профили безопасности -> Контроль приложений** создайте профиль, который разрешает почтовые протоколы, а остальное запрещает.

3\. Создайте разрешающее правило FORWARD. Выберите созданный в шаге 1 профиль, включите правило и нажмите **Добавить**.

4\. Создайте запрещающее весь трафик правило Файрвола. В таблице FORWARD поместите созданное разрешающее правило выше запрещающего.
