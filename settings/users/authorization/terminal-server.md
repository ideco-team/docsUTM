# Авторизация пользователей терминальных серверов

Особенности работы для пользователей терминальных серверов:

* Пользователи имеют собственный IP-адрес, поэтому правила **Файрвола** и **Контроля приложений** можно применить для каждого пользователя.
* Для подключения пользователй необходимо установить на терминальный сервер **Ideco Client**.

## Настройка авторизации пользователей

1\. Скачайте Ideco Client на терминальный сервер. Перейдите в раздел **Пользователи –> Ideco-Client**, переведите опцию **Ideco Client** в положение **Включен**, нажмите **Скачать под Windows**.

![](/.gitbook/assets/terminal-server6.png)

2\. Настройте VPN-подключение терминального сервера через [Ideco Client](settings/users/ideco-client/README.md)

3\. Настройте **Ideco Client** в режиме [Device VPN](settings/users/ideco-client/device-vpn.md). Авторизация появится в разделе **Учётные записи**:

![](/.gitbook/assets/terminal-server7.png)

4\. Перейдите в раздел **Пользователи –> Ideco-Client** на вкладку **Подсети для терминальных серверов** и задайте подсеть для выдачи IP-адресов пользователям.

![](/.gitbook/assets/terminal-server8.png)

* **Название** - укажите название подсети.
* **Устройство** - выберите терминальный сервер.
* **Подсеть** - укажите подсеть в которой пользователям будут выдаваться IP-адреса.
* **Коммантарий** - не обязательное поле.

{% hint style="warning" %}
Подсеть не должна пересекаться с уже имеющейся подсетью, в том числе с той, что используется для VPN-подключений.

У пользователей будут IP-адреса из заданной подпсети, а у сервера будет IP-адрес из VPN-подсети.
{% endhint %}

5\. Вернитесь на терминальный сервер и переключите работу Ideco Client на режим работы терминального сервера.

6\. В разделе **Мониторинг -> Авторизованные пользователи** можно увидеть авторизованные терминальные сервера и пользователей.

![](/.gitbook/assets/terminal-server8.png)
