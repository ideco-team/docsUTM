# Аутентификация

В Ideco NGFW 21.0 аутентификация без указания домена будет доступна только для пользователей из каталога по умолчанию.

### Каталог по умолчанию 

Каталог по умолчанию используется для поиска пользователей, которые аутентифицируются без указания домена. В качестве такого каталога можно выбрать каталог **Локальные пользователи**, интегрированный домен Active Directory, ALD или RADIUS.

Например, если выбрать интегрированный домен из AD в качестве каталога по умолчанию, пользователи смогут авторизоваться с использованием формата `username` вместо `username@<имя домена>`.

![](/.gitbook/assets/authentication.png)

{% hint style="warning" %}
При отключенной интеграции с AL/ALD или RADIUS, всем пользователям придется указывать каталог при авторизации, иначе авторизация не пройдет. Чтобы этого избежать, выберите другой каталог по умолчанию.
{% endhint %}

### Доменный суффикс

Если каталог **Локальные пользователи** не выбран по умолчанию, можно указать доменный суффикс для аутентификации пользователей. Например, при суффиксе `local` локальные пользователи должны будут вводить логины в формате `username@local`:

![](/.gitbook/assets/authentication1.png)

### Алгоритм аутентификации

В таблице **Алгоритм аутентификации** перечислены все доступные каталоги и формат ввода логина для каждого каталога. Поиск пользователей при аутентификации осуществляется в соответствии с расположением каталогов в таблице в порядке приоритета сверху вниз:
